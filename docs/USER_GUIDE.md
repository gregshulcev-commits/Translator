# Руководство пользователя — MVP v9

## Какие части проекта входят в v9

В архив входят две связанные части:

1. desktop-приложение на Python + Tkinter;
2. Android source branch `android-client/`.

Готовый APK в архив **не входит**.

## Установка desktop-приложения

### Рекомендуемый способ

```bash
./install_app.sh
```

После выполнения установщик:

- создаёт `.venv`;
- ставит зависимости;
- создаёт launcher `~/.local/bin/pdf-word-translator-mvp`;
- добавляет приложение в меню Linux с иконкой.

Launcher ссылается на текущую папку проекта. Если вы позже перенесёте исходники в другое место, просто запустите `./install_app.sh` ещё раз из новой директории.

### Если нужен только базовый режим

```bash
./install_app.sh --skip-optional
```

Это полезно, если вы хотите сначала поставить только словарный desktop-режим, а Argos добавить потом уже из GUI.

## Первый запуск

Приложение можно запустить:

- из меню приложений Linux;
- или так:

```bash
~/.local/bin/pdf-word-translator-mvp
```

## Поддерживаемые документы

Desktop умеет открывать:

- PDF;
- TXT;
- FB2.

Для PDF нужен текстовый слой.

## Базовый сценарий desktop

1. Откройте документ.
2. При необходимости переключите направление `EN → RU` или `RU → EN`.
3. Кликните по слову.
4. В нижней панели появится словарная карточка.
5. Если включён provider layer, во второй строке появится короткий контекстный перевод или статус провайдера.

## Поиск по документу

В toolbar есть строка поиска и кнопки:

- `Найти`;
- `Пред.`;
- `След.`.

Toolbar в `v9` сохранён адаптивным, но кнопки изменения UI-шрифта из него убраны: эти настройки перенесены в отдельное окно **«Настройки»**.

## Масштабирование

Поддерживаются:

- кнопки `-` и `+`;
- `Ctrl + колесо мыши`.

Верхний предел масштабирования — **800%**.

## Настройки

Окно **«Настройки»** теперь является основным способом конфигурации.

### Вкладка «Общие»

Здесь настраиваются:

- направление перевода;
- активный контекстный провайдер;
- размер шрифта интерфейса.

### Вкладка «LibreTranslate»

Здесь можно:

- указать URL сервера;
- задать API key;
- проверить оба направления `EN ↔ RU`.

### Вкладка «Yandex Cloud»

Здесь можно задать:

- `Folder ID`;
- `API key`;
- `IAM token`.

### Вкладка «Argos»

Здесь можно без терминала:

- установить Python runtime для Argos;
- обновить индекс моделей;
- скачать модель;
- импортировать локальный `.argosmodel`;
- сразу выбрать Argos активным провайдером.

### Вкладка «Словари»

Здесь можно:

- посмотреть все подключённые словари;
- открыть каталог словарей;
- подключить SQLite;
- импортировать CSV;
- импортировать FreeDict TEI;
- удалить пользовательский runtime-словарь.

Встроенные словари показываются в списке, но из GUI не удаляются.

## Словари

### Каталог словарей

Каталог открывается через:

- кнопку **Словари**;
- меню **Словари**;
- вкладку **Словари** в настройках.

Через каталог можно:

- установить bundled CSV packs;
- скачать и собрать FreeDict packs;
- открыть источник словаря.

Открытие источника ограничено только `http/https` ссылками.

### Удаление словарей

В `v9` удаление пользовательских словарей доступно из GUI.

Что важно:

- удаляются только словари из пользовательской runtime-папки;
- встроенные словари остаются неизменяемыми;
- удаление выполняется с дополнительной проверкой пути, чтобы не затронуть посторонние файлы.

## Контекстный перевод

### Какой контекст отправляется провайдеру

В `v9` приложению больше не передаётся произвольный большой кусок страницы.

Логика стала осторожнее:

- сначала ищется короткое предложение в пределах текущего текстового блока;
- если блок выглядит как таблица, заголовок или контекст получается слишком длинным, берётся только текущая строка;
- это снижает вероятность захвата названия документа, шапки страницы или чужой ячейки таблицы.

### LibreTranslate

Что важно:

- по умолчанию ожидается self-hosted адрес `http://127.0.0.1:5000`;
- можно указать базовый адрес или endpoint `/translate`;
- для публичного `libretranslate.com` нужен API key;
- в окне настроек есть проверка обоих направлений `EN ↔ RU`.

### Yandex Cloud

Нужно указать:

- `Folder ID`;
- `API key` или `IAM token`.

### Argos (офлайн)

Рекомендуемый путь через GUI:

1. откройте **Настройки → Argos**;
2. нажмите **Установить поддержку Argos**;
3. затем **Обновить список из сети**;
4. выберите модель и установите её;
5. нажмите **Использовать Argos**.

Альтернативный путь через отдельное окно:

- **Перевод → Офлайн-модели Argos…**.

Справка **«Как установить Argos…»** сохранена как отдельное read-only окно с кнопкой копирования.

## Настройки и безопасность

Настройки сохраняются в:

```text
~/.local/share/pdf_word_translator_mvp/settings.json
```

Файл:

- сохраняется атомарно;
- на POSIX получает права `0600`.

## External plugins

Внешние Python-плагины по умолчанию **выключены**.

Для включения:

```bash
export PDF_WORD_TRANSLATOR_ENABLE_EXTERNAL_PLUGINS=1
```

## Android source branch

Android-клиент — это отдельный исходный Gradle-проект.

Что уже есть:

- Kotlin UI;
- Chaquopy bridge;
- PdfRenderer;
- bundled SQLite assets;
- `mobile_api.py` для словарного lookup.

Как открыть:

1. запустить Android Studio;
2. открыть `android-client/`;
3. дождаться Gradle sync;
4. собрать `debug` APK из IDE.

Что пока не готово:

- tap-to-word selection;
- OCR;
- финальный APK в архиве.
