# `dictionary_manager.py`

## Назначение

GUI-friendly helper для управления уже установленными словарными пакетами.

В `v9` модуль нужен прежде всего для вкладки «Словари» в новом окне настроек.

## Что делает модуль

### 1. Перечисляет установленные словари

`list_installed_dictionary_records(config)` возвращает список паков с дополнительными признаками:

- bundled или пользовательский;
- путь к SQLite-файлу;
- можно ли безопасно удалить пакет из GUI.

### 2. Безопасно удаляет только runtime-паки

`remove_installed_dictionary(record, config)` разрешает удаление только когда:

- пакет не bundled;
- файл находится внутри пользовательской runtime-папки;
- путь существует и является обычным файлом.

Если условия не соблюдены, модуль поднимает `DictionaryManagerError`.

## Почему это важно

Удаление словарей из GUI не должно давать приложению возможность удалить произвольный файл из файловой системы пользователя. Поэтому helper специально ограничивает операции рамками runtime-папки.

## Что не удаляется

- встроенные bundled словари;
- файлы вне runtime directory;
- небезопасные/несуществующие пути.
