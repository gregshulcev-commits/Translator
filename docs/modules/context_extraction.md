# `context_extraction.py`

## Назначение

Консервативное извлечение короткого контекста вокруг выбранного слова.

Этот модуль появился в `v9`, чтобы контекстный перевод не уносил лишний текст с целой страницы PDF или TXT-документа.

## Основная идея

Вместо старого подхода «идти влево/вправо по странице до пунктуации» модуль использует более осторожную эвристику:

- **не пересекать границу блока**;
- сначала попытаться взять **предложение внутри текущего блока**;
- если блок выглядит как таблица/заголовок или фрагмент слишком длинный — вернуть **только текущую строку**.

## Почему это важно

Такой подход уменьшает риск, что в провайдер уйдут:

- название документа;
- заголовок раздела;
- соседняя ячейка таблицы;
- слишком длинный кусок страницы.

## Основная функция

- `extract_compact_context(sentence_words, token_index, *, max_tokens=40, max_chars=260)`

Она возвращает строку, которую можно безопаснее отправлять в online/offline context provider.

## Где используется

- `document_pdf_pymupdf.py`
- `document_text_base.py`

## Проверка

Для модуля добавлены отдельные тесты, которые проверяют:

- извлечение предложения в пределах блока;
- fallback на текущую строку для табличного блока;
- fallback на строку для слишком длинного предложения.
