# `install_app.sh`

## Назначение

Пользовательский установщик desktop-приложения для Linux.

Это главный ответ на сценарий `v9`: «запустить один файл и получить установленное приложение с launcher и иконкой, без ручного `source` и длинных команд».

## Что делает скрипт

- создаёт `.venv` в корне проекта;
- обновляет `pip`, `setuptools`, `wheel`;
- устанавливает `requirements.txt`;
- по умолчанию пытается установить `requirements-optional.txt`;
- вызывает `tools/install_default_dictionaries.py`;
- создаёт launcher в `~/.local/bin/pdf-word-translator-mvp`;
- создаёт desktop entry в `~/.local/share/applications/pdf-word-translator-mvp.desktop`;
- копирует icon в `~/.local/share/icons/hicolor/256x256/apps/`.

## Параметры

- `./install_app.sh` — базовая установка;
- `./install_app.sh --skip-optional` — установка без optional Argos runtime.

## Что получает пользователь после установки

- запуск из меню приложений Linux;
- отдельный launcher для запуска из терминала;
- отсутствие необходимости вручную активировать виртуальное окружение.

Launcher привязывается к текущей распакованной директории проекта. Если каталог с исходниками будет перенесён, установщик нужно запустить повторно.

## Ограничения

Скрипт ориентирован на Linux desktop и пользовательские каталоги `~/.local/*`.

Для других платформ он может служить шаблоном, но не гарантирует готовую интеграцию без адаптации.
