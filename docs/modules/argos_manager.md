# `argos_manager.py`

## Назначение

Общий helper для lifecycle optional Argos runtime и офлайн-моделей.

Это центральная точка, которую используют:

- `ArgosContextProvider`;
- GUI manager в `main_window.py`;
- CLI helper `tools/install_argos_model.py`.

## Что решает модуль

### 1. Проверяет наличие optional runtime

Если `argostranslate` не установлен, модуль не роняет приложение, а возвращает мягкий статус с подсказкой, как поставить optional dependency.

### 2. Показывает состояние поддерживаемых направлений

Сейчас модуль ведёт состояние для:

- `EN → RU`
- `RU → EN`

Для каждого направления возвращается:

- установлена ли модель локально;
- доступна ли модель для скачивания;
- версия пакета;
- имя пакета;
- поясняющий status text.

### 3. Скачивает и ставит модель из индекса

`install_argos_model_for_direction(direction)`:

- обновляет индекс Argos;
- находит пакет нужного направления;
- скачивает `.argosmodel`;
- вызывает `argostranslate.package.install_from_path()`.

### 4. Импортирует локальный `.argosmodel`

`import_argos_model_from_path(path)` ставит уже скачанный архив и затем пересчитывает состояние направлений.

### 5. Проверяет готовность направления к переводу

`argos_direction_ready(direction)` — компактная проверка для провайдера:

- если runtime не установлен, вернёт `False` и понятное сообщение;
- если модели нет, вернёт `False` и подсказку открыть менеджер офлайн-моделей;
- если всё готово, вернёт `True`.

## Почему это вынесено отдельно

Без такого helper-модуля GUI, CLI и provider layer начали бы дублировать одну и ту же optional-логику:

- dynamic import `argostranslate`;
- проверку индекса;
- поиск пакета по направлению;
- обработку отсутствующей модели.

Выделение в отдельный модуль делает поведение согласованным и удобным для тестирования.
