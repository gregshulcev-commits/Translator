# Оценка ресурсов для нейронного перевода

## Короткий вывод

Для этого проекта нейронный перевод лучше рассматривать как **optional plugin / language pack**, а не как часть ядра.

Причины:

- модели EN↔RU занимают сотни мегабайт каждая;
- для двух направлений диск легко выходит за 1 ГБ;
- на слабых устройствах RAM и latency становятся заметными.

## Практическая оценка

### Marian / OPUS-подобные модели

Для EN→RU и RU→EN у распространённых open-source Marian/OPUS моделей типичные величины такие:

- один `pytorch_model.bin`: около **307 MB**;
- один `rust_model.ot`: около **563 MB**;
- полный репозиторий модели с несколькими форматами весов и токенайзерами: около **875 MB – 1.49 GB** в зависимости от версии и набора файлов.

Если брать **runtime-only веса** для одного направления, реалистично ориентироваться примерно на:

- **350–600 MB на диск** за одно направление;
- **700 MB – 1.2 GB на диск** за два направления.

### RAM

Грубая безопасная оценка для desktop:

- **0.8–1.5 GB RAM** на одну загруженную модель во время инференса;
- больше, если держать сразу оба направления или если использовать тяжёлый runtime.

### Android

Для Android это особенно чувствительно:

- язык нужно скачивать отдельно;
- устройство должно быть 64-bit / aarch64 для ряда on-device translation сценариев;
- даже при работе офлайн storage и RAM будут заметным ограничением.

## Что это значит для проекта

### Desktop (ThinkPad T14 Gen 2, 16 GB RAM)

Нейронный перевод как плагин — **реалистичен**.

- EN→RU-only plugin: нормальный сценарий;
- EN↔RU plugin: тоже возможен, но лучше как optional install.

### Android (Poco M5, 6 GB RAM)

Нейронный перевод возможен только очень осторожно:

- одна language pair за раз;
- отдельная установка языкового пакета;
- агрессивное освобождение модели из памяти;
- желательно sentence-level translation по кнопке, а не постоянный background lookup.

## Рекомендация

Лучший путь для этого проекта:

1. ядро остаётся словарным;
2. нейронный перевод добавляется как **plugin layer**;
3. модель ставится отдельно через GUI/installer;
4. по умолчанию нейронка выключена.
