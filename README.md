# Офлайн переводчик PDF по клику на слово — MVP

## Что это

Это рабочий MVP настольного приложения для Linux, которое:

- открывает PDF с текстовым слоем;
- показывает страницу документа;
- позволяет кликнуть по слову мышью;
- выделяет выбранное слово;
- показывает перевод справа в постоянной панели;
- показывает транскрипцию, альтернативные переводы и контекст из документа;
- умеет искать по документу;
- работает без сети после установки.

## Почему текущий MVP реализован на Python/Tkinter + PyMuPDF

На этапе проектирования обсуждалась архитектура на Qt/C++. Для архива этого MVP выбран более быстрый путь к работающему результату:

- **Python 3** — минимальный порог входа и быстрая итерация;
- **Tkinter** — стандартный GUI toolkit, доступный в типовой Linux-среде;
- **PyMuPDF** — быстрый PDF-движок с доступом к словам и их координатам.

При этом архитектура сохранена **модульной**:

- документ открывается через плагин документа;
- перевод идет через плагин словаря;
- UI не знает внутренностей PDF-парсера;
- словарь можно заменить на более полный без переписывания viewer-а.

Такой вариант полезен как **первый рабочий срез**, на котором уже можно читать реальные PDF и продолжать разработку без переписывания с нуля.

## Что входит в MVP

- поддержка **PDF**;
- поддержка **text-based PDF** (без OCR);
- перевод **EN -> RU**;
- правая закрепленная панель перевода;
- поиск по документу;
- стартовый локальный словарь;
- документация по модулям, форматам и тестам;
- автоматические тесты и smoke-проверка GUI.

## Что не входит в MVP

- OCR;
- перевод текста внутри изображений;
- DOCX/EPUB;
- история слов;
- нейронный перевод;
- Android-сборка;
- полноценный большой словарь общего назначения.

## Ограничение стартового словаря

В архив включен **стартовый словарь**, ориентированный на технические документы и приложенные тестовые PDF. Его задача — сделать MVP полностью автономным и воспроизводимым внутри архива.

Архитектура уже подготовлена к замене этого словаря на более полный:

- через новый CSV;
- через подготовленный SQLite-файл;
- через будущий импортёр внешних словарей.

## Быстрый запуск

### Вариант 1. Прямой запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
PYTHONPATH=src python3 -m pdf_word_translator.app
```

### Вариант 2. Через CMake

```bash
cmake -S . -B build
cmake --build build --target run
```

### Запуск с PDF сразу

```bash
PYTHONPATH=src python3 -m pdf_word_translator.app /path/to/file.pdf
```

## Сборка стартового словаря заново

```bash
PYTHONPATH=src python3 tools/build_dictionary.py data/starter_dictionary.csv data/starter_dictionary.sqlite
```

## Запуск тестов

```bash
PYTHONPATH=src pytest
xvfb-run -a python3 tests/smoke_gui.py
```

## Реализованный пользовательский сценарий

1. Пользователь открывает PDF.
2. Приложение рендерит страницу и получает координаты слов.
3. Пользователь кликает по слову.
4. Сервис находит `WordToken` по координате.
5. Панель справа показывает:
   - выбранное слово;
   - словарную форму;
   - транскрипцию;
   - лучший перевод;
   - альтернативы;
   - контекст из документа.

## Точки расширения

- новый плагин документа (`DOCX`, `EPUB`, `OCR-page-as-text`);
- новый словарь `SQLite/CSV/StarDict importer`;
- история слов;
- отдельный OCR-плагин;
- отдельный нейронный переводчик фраз и предложений.
