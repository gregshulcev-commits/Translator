# Известные ограничения

## Документы

- desktop PDF должен содержать текстовый слой;
- OCR отсутствует;
- текст внутри изображений не переводится;
- TXT / FB2 рендерятся как synthetic pages.

## Viewer (desktop)

- lazy rendering уменьшает фризы, но очень тяжёлый page render всё ещё может ощущаться на больших PDF и высоких масштабах;
- thumbnails и overview-панель страниц пока отсутствуют.

## Словари

- качество перевода зависит от состава и порядка подключённых паков;
- FreeDict зависит от наличия сети на этапе установки;
- нет удаления словарей из GUI, только добавление / подключение.

## Argos и provider layer

- словарный перевод слова работает офлайн;
- контекстный перевод зависит от выбранного optional провайдера;
- Yandex / LibreTranslate требуют внешнюю настройку и сеть;
- Argos требует отдельной установки optional dependency и модели;
- Argos manager сейчас ориентирован на направления **EN ↔ RU**;
- удаление Argos-моделей из GUI пока не реализовано.

## UI (desktop)

- интерфейс построен на Tkinter, поэтому остаётся прагматичным desktop shell, а не fully modern native shell;
- настройка интерфейса сейчас ограничена размером шрифта;
- установка Argos-модели из GUI пока без отдельного progress bar.

## Платформы

- desktop MVP протестирован как Linux desktop приложение;
- Android ветка в v7 включена как **исходный проект**, а не как уже собранный APK;
- в архив не включён `gradlew` wrapper;
- финальная Android сборка должна делаться в Android Studio или локальном Gradle-окружении с установленным SDK.

## Android branch v7

- PDF уже рендерится, но tap-to-word selection ещё не реализован;
- lookup в Android-клиенте пока запускается по **ручному вводу слова**;
- контекстный provider layer пока не подключён к Android UI;
- Argos model manager пока существует только в desktop-клиенте.
