# V5 release notes

## Исправления

- Убрано принудительное возвращение viewport к выделенному слову или найденному совпадению при обычной перерисовке overlay. Теперь после выделения слова можно спокойно листать документ дальше.
- Исправлено отображение строк в каталоге словарей при увеличении размера интерфейса: `Treeview` теперь пересчитывает `rowheight` по текущему размеру шрифта.
- Переделана доставка результатов контекстного перевода в UI: результаты из фонового потока попадают в очередь и применяются в главном Tk-потоке.
- Устранена скрытая гонка с синхронным callback при пустом контексте: активный `request_id` назначается до запуска перевода.
- Добавлена отмена устаревших запросов контекстного перевода при очистке панели, смене направления и смене провайдера.
- `SettingsStore` стал устойчивым к лишним ключам и некорректным значениям в `settings.json`.
- Для `Yandex Cloud` `Folder ID` теперь валидируется как обязательный параметр перед сетевым запросом; UI и документация синхронизированы с этим требованием.

## Небольшие улучшения производительности

- Для PDF/TXT/FB2 добавлен индекс `token_id -> position` при сборке контекста предложения. Это убирает линейный поиск позиции токена при получении sentence context.
