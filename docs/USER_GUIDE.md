# Пользовательское руководство

## Какие части проекта есть в v7

В архиве теперь две ветки:

1. **desktop-приложение** — основная рабочая версия;
2. **android-client** — начальная APK-ветка с отдельным UI.

Если вам нужен текущий полный функционал, используйте desktop-клиент.

## Desktop: поддерживаемые документы

Desktop-приложение умеет открывать:

- PDF с текстовым слоем;
- TXT;
- FB2.

## Desktop: открытие документа

1. Нажмите **Открыть** на панели инструментов.
2. Либо используйте **Файл → Открыть документ…**.
3. Выберите `*.pdf`, `*.txt` или `*.fb2`.
4. После загрузки документ появится в общем scrolling viewer-е.

## Desktop: перевод слова

1. Прокрутите документ к нужному месту.
2. Нажмите на незнакомое слово.
3. Слово выделится оранжевой рамкой.
4. Внизу появится компактная справка:
   - форма слова / транскрипция;
   - лучший перевод;
   - контекстный перевод или пример;
   - альтернативные переводы.

Если перевод не найден, приложение покажет проверенные candidate forms.

## Desktop: направление перевода

Переключить направление можно:

- кнопкой `EN → RU` / `RU → EN` на toolbar;
- через меню **Перевод → Направление**.

Смена направления сразу влияет на следующие клики. Если слово уже выделено, приложение переоценит перевод для текущего выбора.

## Desktop: прокрутка и zoom

### Прокрутка

- колесо мыши — прокрутка документа вверх / вниз;
- кнопки `◀` и `▶` — переход к предыдущей / следующей странице;
- `PageUp` / `PageDown` — переход между страницами.

После исправлений v5/v6 viewer больше не возвращает вас к старому highlight при обычной прокрутке.

### Масштабирование

- кнопки `-` и `+` на toolbar;
- `Ctrl + колесо мыши` — быстрый zoom под курсором.

Текущий лимит масштаба: **50% — 800%**.

## Desktop: контекстный перевод

Во второй строке нижней панели приложение может показывать контекстный перевод предложения.

Режим выбирается в меню **Перевод → Контекстный перевод**:

- `Отключено`;
- `Argos (офлайн)`;
- `LibreTranslate`;
- `Yandex Cloud`.

### LibreTranslate

Для `LibreTranslate` укажите URL сервера и, при необходимости, API key.

### Yandex Cloud

Для `Yandex Cloud` заполните:

- `Folder ID`;
- `API key` **или** `IAM token`.

### Argos (офлайн)

Для `Argos` нужны две вещи:

1. установленный Python runtime `argostranslate`;
2. установленная модель нужного направления.

Если чего-то не хватает, desktop-приложение показывает понятную подсказку в нижней панели и в окне настроек провайдера.

## Desktop: офлайн-модели Argos из GUI

### Где открыть

Меню:

- **Перевод → Офлайн-модели Argos…**

Окно показывает:

- направление (`EN → RU`, `RU → EN`);
- статус модели;
- версию найденного пакета;
- имя пакета Argos;
- пояснение, что именно ещё нужно сделать.

### Сценарий 1. Скачать модель из сети

1. Установите optional dependency:

```bash
source .venv/bin/activate
python -m pip install -r requirements-optional.txt
```

2. Откройте **Перевод → Офлайн-модели Argos…**.
3. Нажмите **Обновить список из сети**.
4. Выберите нужное направление.
5. Нажмите **Установить выбранную модель**.
6. Нажмите **Выбрать Argos** или переключите провайдер вручную.

### Сценарий 2. Импортировать локальный `.argosmodel`

1. Откройте **Перевод → Офлайн-модели Argos…**.
2. Нажмите **Импортировать .argosmodel…**.
3. Выберите локальный файл.
4. Переключитесь на `Argos (офлайн)`.

### Сценарий 3. Быстрая справка

Пункт **Перевод → Как установить Argos…** теперь открывает **растягиваемое окно справки**, а не узкий message box.

Что это даёт:

- текст нормально расширяется вместе с окном;
- длинные команды читаются заметно лучше;
- команды можно **скопировать кнопкой**.

## Desktop: поиск

1. Введите слово или фразу в строку `Поиск`.
2. Нажмите `Найти`.
3. Используйте `Пред.` и `След.` для переключения между совпадениями.
4. Найденный фрагмент подсвечивается зелёной рамкой.

## Desktop: управление словарями

### Каталог словарей

Откройте **Словари → Каталог словарей…**.

Каталог показывает:

- название пакета;
- направление;
- категорию;
- источник;
- описание.

Из окна каталога можно:

- установить выбранный пакет;
- открыть страницу источника в браузере;
- обновить список.

### Импорт словарей

Меню **Словари** и правый клик позволяют:

- подключить готовый SQLite-пак;
- импортировать CSV-глоссарий;
- импортировать FreeDict TEI;
- установить FreeDict для текущего направления.

## Desktop: изменение размера интерфейса

Используйте кнопки `A−` и `A+` на toolbar или пункты меню **Вид**:

- увеличить размер интерфейса;
- уменьшить размер интерфейса;
- сбросить размер к значению по умолчанию.

Настройка сохраняется автоматически.

## CLI-помощники

### Проверить и поставить Argos модели

```bash
source .venv/bin/activate
PYTHONPATH=src python tools/install_argos_model.py --list
PYTHONPATH=src python tools/install_argos_model.py --from-lang en --to-lang ru
PYTHONPATH=src python tools/install_argos_model.py --file /path/to/model.argosmodel
```

### Импорт словаря

```bash
PYTHONPATH=src python tools/import_dictionary.py my_glossary.csv ~/.local/share/pdf_word_translator_mvp/dictionaries/my_glossary.sqlite --format csv
```

## Android branch v7

Android-клиент пока является **прототипом APK-ветки**, а не полноценной заменой desktop-приложения.

### Что уже умеет Android-клиент

- открыть PDF через системный picker;
- показать PDF-страницу;
- листать страницы;
- ввести слово руками и получить словарный перевод;
- переключить `EN → RU` / `RU → EN`.

### Что пока не умеет Android-клиент

- выбрать слово тапом прямо по PDF;
- показать тот же полный desktop workflow клика по слову;
- управлять Argos-моделями через Android GUI;
- собираться из этого архива «в один клик» без Android Studio/SDK.

### Как открыть Android-ветку

1. Откройте `android-client/` в Android Studio.
2. Дождитесь Gradle sync.
3. Соберите `debug` APK.

## Что ещё пока ограничено

- OCR отсутствует;
- desktop PDF должен иметь текстовый слой;
- Argos сейчас встроен как EN↔RU ветка;
- удаление словарей и Argos-моделей из GUI пока не реализовано;
- Android branch пока является исходным проектом, а не готовым APK-файлом.
